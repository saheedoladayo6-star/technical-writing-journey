# Markdown Links and Images

## Introduction

Links and images make documentation more useful and easier to understand.

## Creating a Link

Syntax:

```markdown
[OpenAI](https://openai.com)
```

Example:

[GitHub](https://github.com)

## Adding an Image

Syntax:

```markdown
![Alt text](image.png)
```

Example:

If you have an image named `screenshot.png` in your project folder:

```markdown
![VS Code Screenshot](screenshot.png)
```

## Best Practices

- Use meaningful link text.
- Always include alt text for images.
- Keep links updated.

## Conclusion

Links help readers navigate documentation, while images make instructions easier to follow.
